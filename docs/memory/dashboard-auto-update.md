# Dashboard Auto-Update Guide

## Problem
The memory dashboard displays static data embedded in HTML. After memory changes, you need to manually run `update-dashboard.sh` to see updates.

## Solutions Implemented

### 1. Automatic Hook (Active) âœ…

**What it does:** Automatically regenerates the dashboard HTML after every agent execution.

**Location:** `.kiro/hooks/auto-update-dashboard.json`

**How it works:**
- Triggers on `agentStop` event
- Runs `bash scripts/update-dashboard.sh`
- Updates dashboard HTML with latest memory data
- Timeout: 10 seconds

**Usage:** No action needed - works automatically!

### 2. Manual Refresh Button

**What it does:** Reloads the page to show updated data.

**How to use:**
1. Click the ðŸ”„ button in the dashboard header
2. Page reloads with fresh data

**Note:** This only works if the HTML was already regenerated by the hook or manual script execution.

### 3. Script Commands

#### Update and Open Dashboard
```bash
bash scripts/open-dashboard.sh
```
Updates the dashboard HTML and opens it in your browser.

#### Update Only (No Browser)
```bash
bash scripts/open-dashboard.sh --update-only
```
Updates the dashboard HTML without opening the browser.

#### Direct Update
```bash
bash scripts/update-dashboard.sh
```
Regenerates the dashboard HTML file.

## Workflow

### Automatic (Recommended)
1. Work with AgentX
2. AgentX updates memory files
3. Hook automatically regenerates dashboard
4. Open dashboard: `bash scripts/open-dashboard.sh`
5. See latest data immediately

### Manual
1. Make changes to memory files
2. Run: `bash scripts/update-dashboard.sh`
3. Refresh browser or click ðŸ”„ button

## Troubleshooting

### Dashboard shows old data
**Solution:** Run `bash scripts/update-dashboard.sh` manually

### Hook not working
**Check:**
1. Hook exists: `ls .kiro/hooks/auto-update-dashboard.json`
2. Hook is enabled in Kiro
3. Check Kiro logs for hook execution

### Browser cache issues
**Solution:** 
- Hard refresh: Ctrl+Shift+R (Linux/Windows) or Cmd+Shift+R (Mac)
- Or click the ðŸ”„ button (adds timestamp to URL)

## Configuration

### Disable Auto-Update Hook
If you want to disable automatic updates:
1. Open Kiro Hook UI (Command Palette â†’ "Open Kiro Hook UI")
2. Find "Auto-Update Memory Dashboard"
3. Disable or delete it

### Change Update Frequency
The hook runs after every agent execution. To change this:
1. Edit `.kiro/hooks/auto-update-dashboard.json`
2. Change `eventType` to:
   - `promptSubmit` - Before agent starts
   - `fileEdited` - When files are saved
   - `userTriggered` - Manual trigger only

## Performance

- Dashboard generation: ~1-2 seconds
- Memory size: ~8-30KB (with skills-on-demand)
- No performance impact on agent execution

## Best Practices

1. **Let the hook work** - Don't manually update unless needed
2. **Use `--update-only`** - When scripting or testing
3. **Keep dashboard open** - Click ðŸ”„ to refresh instead of reopening
4. **Check timestamps** - Dashboard shows "Last Updated" time
